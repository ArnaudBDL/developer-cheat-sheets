# Copilot Studio : Instructions

## Instruction Structure

```text
Role and purpose
Audience
Supported tasks
Source precedence
Tool-use rules
Safety constraints
Escalation conditions
Output expectations
Missing-information behavior
```

## Example

```text
Answer employee policy questions using only approved knowledge sources.
Cite the source used for each policy statement.
Never invent dates, approvals or entitlements.
If evidence is missing, say so and direct the user to the HR service channel.
Ask for confirmation before invoking any tool that changes a record.
```

Keep instructions concise, unambiguous and testable. Do not use instructions as the only authorization control. Re-evaluate them whenever knowledge, tools, channels or orchestration behavior changes.
