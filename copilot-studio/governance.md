# Copilot Studio : Governance

## Governance Areas

```text
Business ownership
Environment strategy
Maker access
Data policies
Approved connectors and models
Agent inventory
Risk classification
Testing gates
Publishing and sharing
ALM
Monitoring and compliance
Retirement
```

## Operating Model

```text
Define standards
Provide approved templates
Review high-risk designs
Automate compliance checks where supported
Gate production releases
Track owners and dependencies
Monitor usage and incidents
Review and retire stale agents
```

Use layered environments, guardrails and approval workflows to distinguish experimentation from production. Keep accountability with named business and technical owners throughout the agent lifecycle.
