# Copilot Studio : Environments

## Environment Purpose

```text
Separate makers, data, connections, solutions and deployed components.
Support development, test and production boundaries.
Apply environment-level access and data policies.
```

## Design Checks

```text
Owner and administrator group
Region and data residency
Developer access
Production access
Data policies
Connections and credentials
Capacity and licensing
Retention and audit requirements
```

Use separate environments where credentials, access controls, lifecycle stage or compliance requirements differ. Avoid direct production authoring, and document who can create, share, import, export and publish agents.
