# OWASP : Security Configuration

## Baseline

- Maintain a repeatable hardened configuration for every environment.
- Disable unused services, endpoints, methods, features and sample content.
- Remove default accounts and rotate default credentials.
- Apply secure headers and restrictive cross-origin policies.
- Disable detailed production errors and directory listings.
- Keep frameworks, runtimes, servers and dependencies updated.
- Separate development, test and production configuration.
- Validate configuration automatically during build and deployment.

## Review Checklist

```text
Default credentials
Debug mode
Error disclosure
Administrative interfaces
Unused ports and services
Cloud storage exposure
CORS policy
Security headers
TLS configuration
File permissions
Container privileges
Infrastructure drift
```
