# OAuth 2.0 : PKCE

## Purpose

PKCE binds an authorization request to the client instance that initiated it, protecting the authorization code exchange against code interception and injection.

## Values

```text
code_verifier   High-entropy secret generated by the client
code_challenge  BASE64URL(SHA256(code_verifier))
method          S256
```

## Authorization Request

```text
code_challenge=CHALLENGE
code_challenge_method=S256
```

## Token Request

```text
code_verifier=ORIGINAL_VERIFIER
```

## Rules

- Generate a new verifier for every authorization request.
- Keep the verifier local to the client transaction.
- Use `S256` rather than the plain transformation.
- Validate challenge and verifier at the authorization server.
- Use PKCE for public and confidential authorization-code clients.
