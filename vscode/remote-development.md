# Visual Studio Code : Remote Development

## Remote Options

```text
Dev Containers
Remote - SSH
WSL
Remote Tunnels
GitHub Codespaces
VS Code for the Web
```

## Model

```text
Local VS Code UI
-> remote extension host
-> remote filesystem, tools, terminal and debugger
```

## Rules

- Install remote-development extensions from approved sources.
- Review host keys, container definitions and forwarded ports.
- Keep credentials in supported remote secret mechanisms.
- Distinguish local extensions from extensions installed in the remote environment.
- Use profiles to tailor remote workflows.
- Verify workspace trust before executing remote tasks or code.
